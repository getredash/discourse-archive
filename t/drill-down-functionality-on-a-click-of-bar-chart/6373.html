Drill down support

opened 12:49PM - 28 Oct 18 UTC
arikfr
Frontend enhancement Feature: Parameters Feature: Dashboards

Today the drill down possibilities in Redash are very limited. Practically you c…an only create drill downs from a Table visualization utilizing the Link column type. To have wider range of drill down scenarios we need to implement: 1. Support for defining the behavior of click events in visualizations (#3012) 2. Optional Parameters Support (#3011) Optional parameters are not mandatory to support drill down scenarios, but it will make them easier as you will be able to reuse a single dashboard for more drill down scenarios by having optional filtering on it enabled based on users input.

You can approximate this behavior using Tables with links. But that might not work for your needs.

I’ve made a solution by linking other queries/dashboards in a table. I’ve used PostgreSQL datasource and format() function to insert a HTML <a href> tag. It’s not a dynamic solution, but if you have only a handful of drilldowns, than it’s viable in small scale.
