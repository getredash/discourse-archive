I don’t have an immediate answer for your. But a few observations and some questions:

  • Why is your scheduled_worker WORKERS_COUNT only 1? How many scheduled queries are you trying to execute?
  • I notice your RQ Status > Queues tab doesn’t even show a scheduled queue…

A common issue with k8s (which we don’t official support, yet) deployments is if you duplicated your redis pod. Redash is built to work on a single message queue. All other pods should point to the same redis, otherwise you’ll have side-effects just like this.

I wonder if your scheduler has been configured to with the right redis URI.

Do you see anything in your logs from the scheduler?

The queue is created the first time it’s used. For example when you first load Redash it won’t even have a queries queue until you try executing your first query. See here. There is nothing special about a Queue in basic terms. It’s just a prefix to the Redis key which rq uses when handing jobs off to workers.

You can test this for yourself by connecting to the redis service and flushing all its keys:

docker-compose run --rm redis redis-cli -h redis

# then..

flushall

…then visit the System Status page and observe that all but the default queue disappeared.

From this we can presume the reason you don’t see a scheduled query queue is because the background execution does not occur.
