jesse April 24, 2020, 3:22pm 2

Redash’s CSP headers block this kind of embedding. You can modify the flask-talisman settings if you wish. But beware it poses security risks for your Redash instance as the embedding page can capture sensitive information from your users if they are tricked into logging into the instance from a page that you don’t maintain.

If your users need the query editor, why not redirect them to a new tab with Redash running? From your screenshot it looks like you’re just embedding a dashboard which is possible without trying to embed the entire application (and recommended).

In redash/settings/__init__.py, there are many settings which prevent from getting it embedding it into another application. We made it work by setting blank to environ settings - REDASH_CONTENT_SECURITY_POLICY and REDASH_FRAME_OPTIONS - as below. Then restart redash.

CONTENT_SECURITY_POLICY = os.environ.get(
    "REDASH_CONTENT_SECURITY_POLICY",
    "",
)

FRAME_OPTIONS = os.environ.get("REDASH_FRAME_OPTIONS", "")
