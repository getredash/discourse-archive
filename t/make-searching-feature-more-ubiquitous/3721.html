Make "searching" feature more ubiquitous

Feature Requests
feature-request
richardcane April 29, 2019, 1:55pm 1

The search box on the Table widget for queries is simple but incredibly useful. It would be good to make this a bit more ubiquitous.

Best major example would be to enable dynamic search filtering in dashboards. Obviously this might come with a performance warning depending on implementation but, for a lot of the stuff we’d find that useful for, our queries take less than 10 seconds.

A lesser but still useful example is to be able to have a searchable query based dropdown list. We have a dropdown of users in a specific context which we need to scroll through. It would be quicker to type it.
